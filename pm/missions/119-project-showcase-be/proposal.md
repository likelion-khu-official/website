## [백엔드] 프로젝트 쇼케이스 — 목록·상세 API + 계약

부원들이 만든 프로젝트를 소개하는 페이지의 백엔드예요. 프로젝트 목록(`/projects`)이랑 상세(`/projects/:id`)를 만들고, 프론트가 바로 붙일 수 있게 `shared/` 계약까지 정리해주세요.

> **왜 찬욱님께 부탁드리냐면** — 이건 원래 백엔드 팀 일인데, 지금 선우님·시현님이 각각 세 개씩(#87·#96·#117, #87·#96·#115) 미션을 맡고 있어서 여유가 없어요. 찬욱님이 백엔드도 하실 수 있다고 해서, 이번 프로젝트 쇼케이스는 잠깐 맡아주시는 거예요. 평소엔 인프라 쪽 일이시니 이번이 특별한 경우라는 것만 알아주세요.

---

**Target** — 이 미션이 전진시키는 기능 트리 노드예요: [`프로젝트 쇼케이스`](https://github.com/likelion-khu-official/website/blob/dev/pm/features/프로젝트-쇼케이스.md)랑 그 아래 [목록](https://github.com/likelion-khu-official/website/blob/dev/pm/features/프로젝트-목록.md)·[상세](https://github.com/likelion-khu-official/website/blob/dev/pm/features/프로젝트-상세.md). 정식 스펙은 [서비스 위키 — 기능 명세](https://github.com/likelion-khu-official/website/wiki/기능-명세)에 있어요.

**Deliverable**
- 프로젝트 목록(`/projects`)·상세(`/projects/:id`) API
- 멤버가 본인이 참여한 프로젝트를 등록·수정·삭제하는 API
- **`shared/types/project.ts` 계약 파일** — 프론트가 이걸 보고 바로 화면을 만들 수 있게, 다른 계약 파일(`feed.ts`·`member.ts`)이랑 같은 스타일로 정리해주세요
- 리뷰용 PR을 dev로 올려주세요

> [!IMPORTANT]
> **Why** — 지금 프로젝트 쇼케이스는 스펙만 있고 아무것도 만들어진 게 없어요. "멋사가 진짜 뭔가 만드는 동아리"라는 걸 보여주는 페이지라, 여기가 채워져야 사이트가 훨씬 완성돼 보여요.

**Done**
- [ ] 프로젝트 목록 API가 대표 이미지·제목·한 줄 소개·기수·스택을 내려준다 (`/projects`)
- [ ] 프로젝트 상세 API가 이미지 전부·참여 멤버(이름·파트)·기수·개발 기간·기술 스택·GitHub 링크를 내려준다 (`/projects/:id`)
- [ ] 로그인한 멤버가 본인 참여 프로젝트를 등록·수정·삭제할 수 있다 (참여하지 않은 프로젝트는 못 건드린다)
- [ ] 관리자가 문제 있는 프로젝트를 숨길 수 있다
- [ ] **`shared/types/project.ts`에 요청·응답 타입이 정의돼 있다**
- [ ] **선우님 또는 시현님의 리뷰 승인 없이는 이 PR을 머지하지 않는다**
