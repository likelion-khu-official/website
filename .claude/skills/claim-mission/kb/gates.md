# 게이트 — 판정은 미션의 Done에서 · 검토자는 채간 사람 본인

게이트에서 잣대를 즉흥으로 만들지 마라. 그 미션의 `Done`(완료기준)과 `Notes`(경계)가 채점표다. 미션마다 다르다 — 정상이다.
**게이트에서 검토하는 사람 = 미션을 채간 그 사람**(PM 아님). 그 사람은 맥락이 거의 없다 → 네가 차려주고 **확인만** 받는다.

## 원칙
- **Done이 유일한 통과 기준.** Done 체크박스가 다 참이 될 수 있으면 통과 후보. 아니면 아직.
- **Notes는 경계.** "하지 말 것 / 다른 팀 접점 / 따라야 할 것"을 어겼으면 통과 못 한다.
- **Done에 없는 걸로 붙잡지 마라.** 내 취향·"더 잘"은 게이트 사유가 아니다. 정말 필요하면 ask-pm.

## 게이트 UX (두 게이트 공통) — 차려서 확인만
1. **㉠ 리뷰포인트를 딱딱 집는다** — 이 미션 Done·Notes 기준 2~4개, 구체적으로.
2. **㉡ 각 포인트에 네 권장 답을 준다** — "제 판단엔 …, 이유는 …".
3. **㉢ "이거 맞아요?"로 확인** — 맞으면 진행, 아니면 어디가 걸리는지만. **열린 질문·시험 금지.**

## RP 게이트 (P 다음 — 방향 승인)
리뷰포인트로 보일 것:
1. R에서 알아낸 핵심(관련 코드·계약·제약).
2. 하려는 접근 + 그게 Done을 어떻게 충족하는지.
3. 불확실/가정(있으면).
→ 각 포인트에 네 권장 판단 → **"이 방향 맞아요?"**
통과 = 방향 OK → 바로 I. 되돌림 = 접근이 Done과 어긋나거나 가정이 틀림 → R/P 보강.

## IQ 게이트 (Q 다음 — 결과 승인 → 마감)
리뷰포인트로 보일 것:
1. **Done 항목별 셀프 QA**(체크리스트, 각 체크박스가 왜 참인지 근거 — 테스트·실행·화면 등).
2. Notes 경계를 안 넘음.
3. 스코프 크리프 없음(Done 밖은 안 건드렸다).
4. 산출물이 어디에 남나(PR·파일·URL).
→ 각 포인트에 네 권장 판단 → **"이대로 마감할까요?"**
통과 = 승인 → **채간 사람이 Step 4에서 마감**(산출물·PR → 보드 Done → 이슈 close). 되돌림 = Done 미충족/경계 위반/크리프 → I로.

## 되돌림은 실패가 아니다
되돌림은 싼 지점에서 방향을 고치는 것이다. 게이트가 존재하는 이유가 그것. 되돌아가면 사유를 명확히 남기고 해당 페이즈부터 다시.
