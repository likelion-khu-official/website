# 모집 알림  🟡

<sup>[⌂ 사이트](README.md) › [공개 사이트](공개-사이트.md) › [모집](모집.md)</sup>

> 평소(모집기간 아닐 때) 방문자가 이메일을 남겨 구독하고, 모집 시작 때 안내 메일을 1회 수동 발송받는 기능.

## 개요
- 평소(모집기간이 아닐 때) 열리는 신청.
- 받는 것 = 이메일 주소 + [필수] 개인정보 수집·이용 동의(딱 이메일 하나 — 신청 문턱 최소화).
- 비로그인 방문자가 신청하고, 관리자는 명단을 열람해 모집이 시작되면 그 명단으로 안내 메일을 한 번 수동 발송(no-reply@likelion-khu.com). 자동 스케줄러 없음.
- 신청 시 이미 있든 없든 "신청됐어요"로 동일 응답(가입 여부가 남에게 드러나지 않게).
- 노출: 랜딩 모집 섹션(평소 "모집 알림 받기").

## 현재 진행 상태
구독의 핵심 한 줄(프론트 폼 → 백엔드 저장)은 완성돼 배포됐다. 프론트가 `shared/`에 API 계약(#41)을 열고, 백엔드가 `POST /api/notifications/subscribe`를 계약대로 구현(#62)했으며, 메인 랜딩의 모집 알림 폼도 이 API에 연동됐다(#42). 다만 아직 **저장까지만**이다 — 어드민의 모집 관리(켜기/끄기·미리보기·메일 발송)와 실제 이메일 발송은 미구현으로, #113 확인 결과 `NotificationSubscriptionService`는 DB 저장만 하고 `EmailService`를 전혀 호출하지 않는다. 발송 인프라·이메일 모듈 자체(OCI Email Delivery, email_log 트래킹)는 어드민 초대/재설정용으로 먼저 구축됐으나(#81·#85) 모집 알림과는 아직 연결돼 있지 않다. 또한 개인정보 수집 동의(#68)·봇 방지(#69)·응답 동일화(#70)는 모두 열린 상태로 남아 있다.
