## [백엔드] — 운영진 소개 관리: 운영진 카드를 만들고 고치는 화면

관리자가 운영진을 지정하고, 그 사람의 소개 카드(직책·학과·학번·사진·소개글)를 만들고 고치고 지우고 순서를 정하는 어드민 화면을 만드는 일이에요. 랜딩의 "운영진 소개" 섹션에 뿌려질 데이터를 사람이 직접 채우고 관리하는 곳이라고 보면 돼요. 이 화면이 쓰는 운영진 소개 API(#87)는 안시현이 이미 만들고 있어서, 그 위에 화면까지 같은 사람이 이어서 붙이는 풀스택 미션이에요.

---

**Target** — 이 미션이 전진시키는 기능 트리 노드는 [`운영진 소개 관리`](../../features/운영진-소개-관리.md)예요(상위 [`어드민`](../../features/어드민.md), 데이터 쪽 관련 노드 [`운영진 소개`](../../features/운영진-소개.md)·[`운영진 카드`](../../features/운영진-카드.md)). 무엇을 담아야 하는지는 이 노드들의 개요가 정본이고, 카드 필드의 정확한 정의는 백엔드 미션 #87 본문에 있어요.

**Deliverable** — `/admin` 안에 들어가는 운영진 소개 관리 화면이에요. 운영진 카드 목록, 새 카드 등록 폼, 기존 카드 수정·삭제, 정렬 순서 조정, 그리고 멤버 명단에서 "운영진 지정"으로 카드 작성을 시작하는 진입점까지요. 이미 만들고 있는 운영진 소개 API(`Staff`, #87) 위에, 그 데이터를 사람이 채우고 고칠 이 관리 화면을 백엔드가 이어서 만들어요. 리뷰용 PR을 dev로 올려주세요.

> [!IMPORTANT]
> **Why it matters** — 백엔드가 운영진 카드 데이터 API(#87)를 만들고 있지만, 그 데이터를 사람이 채우고 고칠 화면이 없으면 API만으로는 아무것도 못 바꿔요. 지금 랜딩 운영진 섹션은 이름도 직책도 플레이스홀더로 박혀 있어요. 이 화면이 있어야 운영진이 개발자 손 안 빌리고 스스로 팀을 갱신하고, 매 기수 새 얼굴로 바로 바꿀 수 있어요. "운영진 지정 → 카드 작성"이 운영진 소개 전체의 입구라, 여기가 막히면 뒷단이 다 준비돼도 랜딩에 아무도 안 떠요. API를 만든 사람이 화면까지 이어서 쥐면 계약과 화면 사이가 벌어질 일이 없어요.

**Done**
- [ ] 관리자가 `/admin`에서 등록된 운영진 카드 목록을 보고, 카드가 정렬값 순서(회장 → 세션장 → 홍보부장 계층이 보이게)대로 나와요.
- [ ] 멤버 명단에서 어떤 멤버를 "운영진으로 지정"하면 그 사람의 운영진 카드를 새로 작성하는 폼으로 이어져요.
- [ ] 카드 작성 폼에서 이름·직책·학과·학번(입학 연도)·사진·소개글·정렬값을 입력해 저장하면, 그 카드가 목록에 나타나요. 사진은 반드시 넣어야 저장되고, 소개글은 비워도 저장돼요.
- [ ] 관리자가 기존 카드를 수정하거나 삭제하면 목록에 곧바로 반영돼요.
- [ ] 관리자가 카드 순서를 바꾸면, 랜딩 "운영진 소개" 섹션에도 그 순서대로 나와요.
- [ ] 최고관리자가 아닌 사람(일반 멤버·로그인 안 한 방문자)은 이 관리 화면에서 등록·수정·삭제를 할 수 없어요.
- [ ] 프론트를 백엔드가 만들었으니, PR 바디에 **프론트엔드 관점 리뷰 포인트**(어떤 화면·상호작용을 어떻게 확인해 달라는지)를 적고 **프론트팀(박일하·김현정)의 리뷰 승인을 받는다.** 이 승인이 없으면 완료로 치지 않는다.

> [!NOTE]
> **Notes**
> - **이 미션은 한 사람이 백엔드와 프론트를 함께(풀스택) 맡습니다.** 어드민은 외부에 안 보이는 내부 운영 화면이라 디자인 의존이 적고, API 계약을 쥔 사람이 화면까지 만들면 프론트↔백엔드를 오갈 필요가 없어서예요. 화면은 예쁨보다 관리자가 쓰기 명확하면 됩니다.
> - **데이터 소스인 운영진 소개 API(#87)가 아직 안 머지됐어요** — PR #129가 리뷰 중이에요. 리소스 이름은 `Staff`로 확정됐고(권한의 admin과 겹쳐서 이름을 바꿨어요), 이 화면은 그 PR의 `Staff` 계약(`shared/types`)과 CRUD·목록 엔드포인트에 붙어요. **#87의 API 계약이 확정·머지된 뒤에 화면을 붙이는** 걸 원칙으로 하되, 계약 형태가 눈에 보이면 화면부터 먼저 만들어두고 나중에 붙여도 괜찮아요. 계약이 흔들리거나 막히면 PM(@xhae123)에게 알려주세요.
> - **운영진 카드는 멤버 API(#76) 데이터와 별개예요.** 멤버 로스터(역할·기수·이모지)랑 합치거나 한쪽으로 대체하지 마세요. 운영진 카드는 공식 정보라 이모지 대체가 없고 **사진이 필수**예요.
> - 이 화면은 "운영진 소개 관리"(어드민 쪽)예요. 랜딩에 카드를 실제로 보여주는 공개 [운영진 소개](../../features/운영진-소개.md) 섹션 구현은 이 미션 밖이에요 — 다만 정렬 결과가 그 섹션에 그대로 반영되는지가 위 Done의 판정 기준이에요.
> - 최고관리자 권한 판정은 어드민 인증·역할(#74/#97)에서 오는 role에 기대요. 화면에서 새로 권한 체계를 만들지 마세요.
> - 사진 업로드는 새로 만들지 말고 이미 있는 이미지 업로드/스토리지 인프라를 재사용하세요.
> - 목록·폼·정렬 UI를 어떻게 짤지는 담당자가 정하면 돼요.
