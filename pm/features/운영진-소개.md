> 랜딩의 "운영진 소개" 섹션에 뿌리는, 멤버 로스터와 별개인 서버 기반 운영진 카드 데이터.

## 하위
- 🟡 [운영진 카드](운영진-카드.md)

## 개요
- 멤버 로스터와 **별개 데이터**의 운영진 카드.
- 필드: 이름(실명) · 직책(회장·부회장·세션장·홍보부장 등 자유) · 학과/학번(학번=입학 연도, 예 22·24) · 사진(필수 — 이 카드의 얼굴이라 이모지 대체 안 함) · 소개글(회장·부회장은 넣고 세션장·홍보부장은 짧거나 없어도 됨) · 정렬 순서(계층이 보이게).
- 지정 흐름: 관리자가 멤버 명단에서 "운영진 지정" → 별도 폼으로 카드 정보 새로 입력 → 노출.
- 공식 정보라 본인이 직접 수정하지 않는다(관리자 관리). 랜딩 "운영진 소개" 섹션에 정한 순서대로 노출.

## 현재 진행 상태
백엔드 착수됐다 — #87(담당 장찬욱, ~7/25)이 운영진 카드 API와 `shared` 계약을 만드는 중이다. 원래 신선우·안시현에게 있었으나 둘 다 다른 미션(#96·#115)으로 여유가 없어 이관됐다. **순서가 정해져 있다**: `shared/types/` 계약만 담은 작은 PR을 먼저 올려 선우·시현 리뷰를 받고 머지한 뒤 나머지 구현을 진행한다 — 계약이 먼저 굳어야 프론트가 붙을 수 있다. PR #97(어드민 인증 파운데이션)이 "운영진 관리"를 담고 있으나, 이는 **어드민 계정** 관리(초대·목록·삭제·역할변경)일 뿐 운영진 소개 카드 데이터와는 별개다.
