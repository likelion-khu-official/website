# 진행 로그 — #8 인프라 · stage/prod 추상화 (프론트가 붙는 서버 + 배포 뼈대)

> PM(김우진)이 **판단·결정한 것만** 시간순으로 추린다. GitHub 코멘트 복붙이 아니라, "왜 이렇게 됐나"의 정제된 흔적.
> 형식: `## YYYY-MM-DD` 아래 일어난 일 / 결정 / 왜.

## 2026-06-27 — 미션 설계 (stage의 주목적 재정의)

- **결정:** stage의 주목적 = "prod 리허설"이 아니라 **프론트가 개발 중 붙는 백엔드 서버**.
  → Done에 "프론트가 stage를 외부에서 호출(공개주소·CORS)"을 핵심으로 박음.
- **왜:** 프론트가 로컬 개발 중 mock 대신 실제 stage 백엔드에 연결해야 한 줄이 진짜로 물림.
  그게 안 되면 프론트가 막혀 — stage가 "항상 떠서 외부에서 닿는" 게 1순위.
- **결정:** 인프라가 stage/prod 복잡성을 **추상화**(개발팀은 "이렇게 올린다"만). + 장찬욱이 stage/prod
  개념을 모를 수 있어 미션에 *개념·왜*와 한국어 입문 자료를 담음 — 떠먹이지 않되 출발은 시킴.
- **결정:** 진짜 배포는 백엔드 의존이라, 이번엔 **더미 컨테이너로 틀**부터. 실앱은 백엔드 크면 합류.
