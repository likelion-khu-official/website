# Outcome 중심 + 로드맵

## Outcome vs Output
- **Output** = 우리가 *내놓는 것*(기능).
- **Outcome** = 우리가 *달성하려는 것*(행동·결과의 변화).
- 기능 로드맵은 "뭘 만들지"를, outcome 로드맵은 "뭘 달성할지"를 말한다. **항상 outcome로.**

## Now / Next / Later (확신을 표현하는 로드맵)
- **Now** — 확정·착수, 성공 지표 정의됨.
- **Next** — 방향성 우선순위. 문제는 알지만 구현은 형성 중.
- **Later** — 전략적 옵션·베팅. 추적만, 확정 아님.
→ 날짜 약속 대신 *확신 수준*을 표현. 전략이 바뀌어도 로드맵이 안정적.

## OKR
Objective(질적 방향) + Key Results(측정 가능한 결과 = outcome). 로드맵 항목이 KR로 연결된다.
