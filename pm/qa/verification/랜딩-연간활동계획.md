## 근거 (시나리오별 상세)

### SYS-007 — pass

`frontend/src/app/activities/page.tsx` 전체를 읽었다:

```tsx
export default function ActivitiesPage() {
  return (
    <main className="min-h-screen flex items-center justify-center">
      <p className="text-gray-400 text-sm">활동 — 준비 중이에요.</p>
    </main>
  );
}
```

`<main>` 안에 `<p>활동 — 준비 중이에요.</p>` 한 줄뿐이고, then이 요구하는 문구("활동 — 준비 중이에요.")와 글자 하나까지 일치한다. `Nav`나 `Footer` import·렌더가 이 파일 어디에도 없다.

Nav·Footer가 전역 레이아웃에서 자동으로 씌워지는 구조가 아닌지도 확인했다 — `frontend/src/app/layout.tsx`(루트 레이아웃)의 `<body>`는 `{children}`만 렌더할 뿐 `Nav`·`Footer`를 포함하지 않는다(랜딩 `page.tsx`가 자기 안에서 직접 `<Nav />`·`<Footer />`를 배치하는 구조, `pm/qa/verification/랜딩-히어로소개통계세션.md`의 LAND-001 참고). 즉 `/activities`가 Nav·Footer 없이 뜨는 건 우연이 아니라, 각 페이지가 자기 레이아웃 요소를 직접 챙기는 구조에서 `activities/page.tsx`가 그걸 아예 안 넣었기 때문이다.

"랜딩에 링크 없음"(given)도 함께 확인했다 — `grep -rn "activities" frontend/src/app/page.tsx frontend/src/components/sections/ frontend/src/components/Nav.tsx`로 찾아봤으나 `/activities` 라우트로 가는 링크는 랜딩 어디에도 없었다(랜딩의 "연간 활동계획" 섹션은 `Plan.tsx`가 `id="plan"` 앵커로만 렌더하는 별개의 정적 섹션이고, `/activities`라는 페이지 자체와는 연결돼 있지 않다). then의 조건(플레이스홀더 문구, Nav·Footer 없음)과 given의 전제(랜딩에 링크 없음) 모두 코드로 확인됐다.
