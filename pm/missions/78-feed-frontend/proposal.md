## [FE] 피드(블로그) — 글 작성·조회 (전체 기능)

### TL;DR

#66 백엔드가 만든 피드 API로, **글을 쓰고 읽는 화면을 통째로** 만들어요. 목록·상세·작성·댓글에 상태·엣지까지 다요. 이번엔 **디자인 시안 없이 기능부터 다 만들고**, 그다음 디자인 피드백으로 다듬어요.

### Why this order (디자인을 나중에)

피드는 **자잘한 기능·상호작용이 많아요** — 매직링크 글쓰기, 이미지 업로드, 댓글, 토큰 상태, 각종 로딩·에러 화면. 이걸 디자인 시안부터 잡으면 끝이 없어요. 그래서 프론트가 동작하는 걸 먼저 세우고, 디자인이 그걸 보고 개선하는 순서로 가요.

### What we're building

**글 조회**
- **글 목록** — 페이지네이션(또는 무한스크롤), 최신순, 목록 카드(썸네일·제목·발췌·작성자·날짜)
- **글 상세** — 본문·이미지·작성자·날짜
- 없는 글은 **404**, 모든 화면에 **로딩·빈·에러** 상태

**글 작성 (매직링크)**
- **토큰 상태 분기** — 매직링크로 들어오면 토큰 상태에 따라 화면이 갈려요: 유효 / 만료 / 이미 사용됨 / 없는 토큰
- **작성 폼** — 제목, 본문, 유효성 검사(필수·길이)
- **이미지 업로드** — 선택·드래그앤드롭, **업로드 진행률**, 미리보기, 삭제, **5MB·확장자 검증**, 실패 처리
- **임시저장(draft)** — 쓰다 나가도 남게
- **작성 중 미리보기**
- **제출** — 중복 제출 방지, 성공/실패 처리, 제출 후 **완료 화면**(1회용이라 다시 못 쓴다는 안내)

**댓글**
- 댓글 조회(+페이지네이션), 익명 작성, 유효성, **빈 상태**, 봇·스팸 대비

**공통 / 횡단**
- **반응형(모바일 우선)**
- **라우팅**(목록·상세·작성 URL)
- **본문 XSS 방어** — 사용자 입력 렌더링 시 이스케이프/새니타이즈
- **외부 공유 미리보기(OG 태그)** — 글이 카톡·링크로 공유되면 미리보기 뜨게
- **이미지 최적화**(lazy load 등)

### Definition of Done

- [ ] 피드에서 공개된 글 목록을 보고, 페이지를 넘길 수 있어요.
- [ ] 글 상세에서 본문·이미지·댓글을 볼 수 있고, 없는 글은 404가 떠요.
- [ ] 매직링크로 들어오면 토큰 상태(유효/만료/사용됨/없음)에 맞는 화면이 떠요.
- [ ] 글(제목·본문·이미지)을 작성해 올릴 수 있고, 이미지 업로드에 진행률·미리보기·검증·실패 처리가 있어요.
- [ ] 임시저장·작성 중 미리보기가 돼요.
- [ ] 제출은 중복이 막히고, 제출 후 완료 화면이 떠요.
- [ ] 익명 댓글을 달고 볼 수 있어요.
- [ ] 모든 화면에 로딩·빈·에러 상태가 있어요.
- [ ] 모바일·데스크탑 반응형이에요.
- [ ] 본문 렌더링에 XSS 방어가 돼요.
- [ ] 글 공유 시 미리보기(OG)가 떠요.

### Notes

- **★ 정해야 할 것 — 본문 포맷.** 플레인 / 마크다운 / 리치에디터 중 뭐로 갈지, 이미지를 본문 중간에 넣을지 첨부만 할지. 작성자가 캐주얼하니 **단순한 쪽 추천**. 착수 때 확정.
- **이번엔 디자인 시안 없이 기능부터.** 스타일은 최소한으로 두고, 디자인팀이 결과물을 보고 개선점을 줘요.
- 백엔드 계약은 `shared/types/feed.ts` (글·댓글 #66, 매직링크·이미지 #73 머지됨).
- **운영진 검수 화면(공개/숨김)은 이번 범위 아니에요** — 어드민(#74)과 별도.
