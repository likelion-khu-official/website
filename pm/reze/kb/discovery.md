# 제품 발견 (Cagan + Torres)

## Empowered Teams (Marty Cagan, SVPG)
- 팀에 **기능 목록이 아니라 풀어야 할 *문제*를 준다.**
- output(배포)이 아니라 **outcome(비즈니스 결과)**에 책임진다.
- 발견 = 솔루션이 **가치있나 · 쓸만한가 · 실현가능한가 · 사업타당한가**(valuable / usable / feasible / viable) 검증.
- 책: INSPIRED · EMPOWERED · TRANSFORMED.

## 지속적 발견 (Teresa Torres, *Continuous Discovery Habits*)
- 고객과 *자주* 접점 → 검증·반복. 3~4번 인터뷰마다 종합·갱신.
- **기회 솔루션 트리(OST):** 목표(outcome) → 기회(고객 pain/need) → 솔루션 → 가정 테스트.
- 효과: 암묵 가정을 명시화하고, *비교·대조* 결정을 하게 함("어디서 이길지").

## PM 적용
솔루션부터 뛰지 마라: **outcome 정의 → 기회(고객 관점) 펼치기 → 그제서 솔루션.** 솔루션 하나에 꽂히면 OST로 대안을 강제한다.
