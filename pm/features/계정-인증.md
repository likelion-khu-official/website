## 개요

**계정 생성**
- 공개 회원가입 없음. 관리자가 부원 계정을 직접 등록한다.
- 아이디 = 학번(전체), 첫 비밀번호 = 전화번호 — 이미 아는 정보라 부원이 따로 낼 게 없다.
- 등록 후 단톡으로 알리고, 부원은 첫 로그인 때 비밀번호를 한 번 바꾼다.

**로그인**
- 학번 + 비밀번호로 들어온다.
- 비밀번호 분실 시 관리자에게 말하면 초기화해 준다(재설정 메일 없음).

**역할 4종**
- 일반 유저(비로그인 방문자) · 멤버(부원) · 관리자 · 최고관리자(딱 1명).
- 권한은 위→아래로만 위임된다. 관리자끼리는 서로 못 준다.

**오프보딩**
- 졸업·탈퇴 계정은 소프트 딜리트 — 로그인만 막고 글·프로젝트·데이터는 그대로 둔다.

서버 기반 동적 영역.

## 현재 진행 상태
운영진(어드민) 인증 파운데이션은 배포돼 동작한다. 백엔드는 이메일=아이디 로그인, JWT를 HttpOnly 쿠키로 발급, 초대(khu.ac.kr 제한·72h 토큰)→가입, 비밀번호 재설정(30분 토큰), 5회 실패 시 계정 잠금, 운영진 목록·삭제·역할변경, "최고관리자 최소 1명" 불변식까지 머지됐고(#97), 그 선행인 이메일 발송 모듈(#85)과 SecurityConfig 골격(#51)도 머지돼 있다. 프론트도 로그인·대시보드 뼈대·초대 수락·비밀번호 재설정 화면이 머지됐다(#99). 이 어드민 파운데이션 위에 멤버 계층이 얹혀 머지됐다(#117, PR #121·#132) — 역할이 2단(SUPER_ADMIN/ADMIN)에서 **4종(일반 유저·멤버·관리자·최고관리자)**으로 넓어졌고, 학번 로그인·전화번호 초기 비번·첫 로그인 강제 비번 변경·관리자 비번 초기화가 코드에 들어왔다. JWT·쿠키·토큰 저장·초대 배선은 어드민 파운데이션 것을 그대로 재사용했다. 프론트 멤버 로그인 화면도 머지됐다(#118, PR #134). **다만 멤버 관리 어드민(등록·발급·정보수정·비번초기화·오프보딩)은 아직 없다** — 그래서 계정을 실제로 발급할 경로가 없어, 로그인·비번 초기화 같은 하위 기능은 코드로는 있어도 운영상 아직 닫히지 않았다(그래서 하위 노드가 🟡). 남은 빈틈은 IP 기반 요청 제한(429), refresh 토큰 로테이션 등으로 `backend/docs/admin-auth-module.md`에 정리돼 있다.
