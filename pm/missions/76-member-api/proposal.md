## [BE] 멤버 API — 멤버 데이터 · 최고관리자 관리

### TL;DR

멤버 정보를 다루는 백엔드 API예요. 멤버를 **만들고, 고치고, 목록을 내려주는 것**까지요. 만들고 고치는 API는 **최고관리자만** 쓸 수 있어요.

### Data

**밖으로 보여지는 정보**
- **이름** — 실명이에요.
- **역할** — PM·FE·BE·디자인·인프라. 겸직도 있으니 여러 개 담을 수 있게요.
- **기수** — 입부 기수예요. 한 번 정해지면 안 바뀌어요.
- **이모지** — 만들 때 랜덤으로 하나 붙여줘요. 그다음엔 안 바뀌고요.
- **사진** — 있으면 이모지 대신 사진을 보여줘요. 없어도 돼요.
- **입부 계기** — 한 줄이에요. 없을 수도 있어요.

**안에서만 쓰는 정보**
- **id**
- **created_by** — 누가 처음 등록했는지.
- **updated_by** — 누가 마지막으로 고쳤는지.
- **created_at / updated_at** — 만든·고친 시각이에요.

> 작은 팁 — "이 사람 입부계기 썼나?"는 `join_reason`이 채워졌는지 보면 돼서, 표시용 필드는 따로 안 만들어요.

### API

- **멤버 만들기** (최고관리자) — 이름·역할·기수 받아서 랜덤 이모지 붙여 저장해요. 누가 만들었는지(`created_by`) 남겨요.
- **멤버 고치기** (최고관리자) — 정보를 바꿔요. 누가 고쳤는지(`updated_by`) 남겨요.
- **멤버 목록** `GET /api/members` (아무나) — 멤버 전체를 내려줘요.

### Definition of Done

- [ ] 최고관리자만 만들기·고치기를 할 수 있어요. 일반 운영진이나 로그인 안 한 사람은 막혀요.
- [ ] 멤버를 만들면 랜덤 이모지가 붙고, 누가 만들었는지 남아요.
- [ ] 멤버를 고치면 누가 고쳤는지 남아요.
- [ ] 멤버 목록을 조회할 수 있어요.
- [ ] 응답 타입이 `shared/types/member.ts`에 정의돼 있어요.

### Notes

- 관리 API의 '최고관리자' 접근은 **#74(어드민 인증)에서 오는 role에 기대요.** #74가 선행이에요.
- 랜덤 이모지는 무난한 것들로만 골라서 줘요(목록은 디자인이랑 맞춰도 되고요).
