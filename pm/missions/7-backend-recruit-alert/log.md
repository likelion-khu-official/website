# 진행 로그 — #7 백엔드 · 모집 알림 저장 API (walking skeleton)

> PM(김우진)이 **판단·결정한 것만** 시간순으로 추린다. GitHub 코멘트 복붙이 아니라, "왜 이렇게 됐나"의 정제된 흔적.
> 형식: `## YYYY-MM-DD` 아래 일어난 일 / 결정 / 왜.

## 2026-06-27 — 미션 설계 (PM이 어디까지 줄지 선 긋기)

- **결정:** 미션에 **무엇을·왜·제약**만 주고 "어떻게"(스키마·엔드포인트·JPA)는 팀에 비움.
  - 박은 것: outcome(모집알림 한 줄 동작) · **SQLite(근거: 단일 인스턴스 셀프호스팅)** · 리소스 제약 · 프론트와 긴밀 협업.
  - 비운 것: 스키마·엔드포인트·저장방식·이식성 해법 → 백엔드 몫.
- **왜:** 이 프로젝트 1순위 가치가 *멤버 학습*. PM이 SQLite "써라"가 아니라 "이래서 SQLite"로 근거를 주면
  팀이 *왜*를 이해하고 *어떻게*를 채운다 — 그게 받아쓰기가 아니라 학습. (PM이 너무 깊이 관여한다는 자각에서 교정)
- **결정:** "왜 SQLite냐"는 인프라(장찬욱)에게 직접 묻게 함 — PM이 팀 간 채널을 독점하지 않게.
