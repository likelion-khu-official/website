## 2026-06-23 — 플랫폼: 오라클 무료티어로 확정

- **결정:** 배포 = 오라클 클라우드 Always Free 단일 노드. 프론트는 Vercel,
  오라클엔 **백엔드 API + DB(셀프호스팅) 함께**(도커컴포즈, 자원 제한 걸고 운영+테스트).
- **왜:** 타사(AWS·GCP·Azure) 동급 컴퓨팅이 월 ~12만원인데 오라클은 0원.
  사용자 적은 초기에 멀티노드는 복잡도·비용만 늘리는 과설계.
- **결정:** 백엔드 언어의 **비용 제약 해제**. Spring(200~300MB)이든
  Go·FastAPI(90MB)든 12GB 단일노드로 운영+테스트 충분 → 메모리 차이 무의미.
  언어는 비용이 아니라 **팀 선호로 결정** → 백엔드 미션(#2)으로 넘김.
- **결정:** dev 브랜치를 main 앞에 두고 테스트 환경용 + protection rule (장찬욱 제안, 승인).

## 2026-06-25 — 팩트체크로 전제 2개 정정 + DB 동거 확정

- **계정 명의:** 동아리 공용이 이상적이나 회비 체크카드가 없음.
  → **김우진 개인 명의**로 가고, 인수인계는 본인이 나갈 때 직접 처리하기로.
- **정정 ①:** 오라클이 **2026-06-15자로** Always Free Ampere A1을
  `4 OCPU/24GB → 2 OCPU/12GB`로 **예고 없이 반토막**. "정책 변경 없는 한 무료"의
  그 변경이 이미 일어남. 우리 워크로드(~1.4GB)엔 12GB로도 충분하나,
  무료티어는 영구 보장이 아님을 실증.
- **정정 ②:** "PAYG 업그레이드가 idle 회수를 막는다"는 **공식 근거 없음**.
  PAYG 효용은 capacity 접근 + 한도 내 무과금뿐. 회수 방어는 별개 →
  **keepalive(7일간 CPU·네트워크·메모리 활동 20% 이상)**로 해결해야 함.
- **결정:** PAYG 안 하고 **무료계정**으로. 한도 초과해도 과금 0(정지)이라
  "절대 돈 안 나감" 보장. 리전은 **서울 1순위, 막히면 도쿄·오사카**(한국에서 30~40ms).
  홈 리전은 계정 생성 시 한 번만 정해지니 그때 확정.
- **DB 동거 → 백업이 필수가 됨:** DB가 같은 노드에 있어 앱과 운명 공유.
  노드 다운·reclaim 시 DB도 함께 멈춤(reclaim은 stop이라 블록볼륨 데이터는 보존,
  단 노드·디스크 장애는 별개). → **정기 DB 덤프 백업**이 옵션이 아니라 필수.
  DB 데이터도 200GB 스토리지를 공유하니 용량 계산에 포함.
- **상태:** 0원 운영 가능 확정. 실비용은 도메인 연 1~2만원(호스팅kr)뿐.

## 2026-06-26 — PAYG로 선회 + 첫 서버 가동

- **전제 번복(무료계정 → PAYG):** 6/25 "무료계정 고수" 결정이 capacity 벽에 막힘.
  도쿄 Ampere가 `OUT_OF_HOST_CAPACITY`로 계속 거부(quota는 남으나 무료계정은
  물리 재고 우선순위가 낮음). → owner(김우진)가 **PAYG로 전환**해 우선순위 확보하니 풀림.
  - **왜 0원 유지:** Always Free 한도 내는 PAYG여도 과금 0(공식). 체험 크레딧 US$300(≈SGD400)도 별도로 남음.
  - **트레이드오프:** 무료계정의 "넘으면 자동 차단"을 잃는 대신 capacity를 삼 → **예산 알림**으로 방어.
- **리전 확정:** 서울이 아니라 **도쿄(`ap-tokyo-1`)**. 홈 리전은 1회 고정이라 이걸로 굳음
  (서울 우선이었으나 신규 무료 계정엔 AD 1개·capacity 빡셈).
- **서버 가동:** `likelion-prod` (Ubuntu 24.04 ARM). capacity 잡기 쉽게 1 OCPU/6GB로
  선점 후, 풀린 뒤 한도 풀(**2 OCPU/12GB**)로 resize. 공인IP `168.138.202.82`. 전부 0원 경계 안.
- **0원 가드레일 하네스화:** 무료 경계(한도·NAT 함정·"available≠무료")와 capacity 운영 팁을
  `infra/CLAUDE.md`에 박음(PR #6). 다음 사람이 같은 벽에 안 부딪히게.
- **권한:** 장찬욱을 IAM Administrators로 위임(키 발급). 계정 owner·결제는 김우진.
