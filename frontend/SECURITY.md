# SECURITY.md — 프론트엔드 보안 규칙

> 상위 문서: [루트 SECURITY.md](../SECURITY.md) — 공통 보안 규칙은 거기, 여긴 프론트엔드에만 특화된 규칙.

## 이 문서는 무엇인가

프론트엔드 작업(화면 구현·API 연동 등)에서만 적용되는 보안 규칙. 아직 비어 있다 — 아직 이 도메인 특화 사고나 결정이 없었기 때문. 채워지기 전에 지레짐작으로 규칙을 만들지 않는다.

## 언제 채워지나

- 이 도메인에서 보안 사고·취약점이 발견되고 재발 방지 규칙이 필요할 때
- 이 도메인에 특화된 보안 결정(예: XSS/CSRF 방어, 클라이언트에 노출 가능한 값의 범위, 인증 토큰 저장 방식)이 내려질 때
- 에이전트가 빠지기 쉬운 이 도메인 특유의 보안 함정을 발견했을 때

항목은 **규칙 + 왜(무슨 사고/근거였는지)** 한 세트로 남긴다. 일반론 금지 — 이 프로젝트에서 실제로 있었던 것만.
