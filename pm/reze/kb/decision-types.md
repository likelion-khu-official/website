# 의사결정 종류 (Bezos)

## Type 1 — 일방향 문 (one-way door)
되돌릴 수 없거나 거의 불가능. 잘못 들어가면 못 돌아옴. → **신중히, 깊게 논의**해서 high-quality 결정.
예: 핵심 아키텍처, 브랜드 정체성, 공개 발행.

## Type 2 — 양방향 문 (two-way door)
되돌릴 수 있음. 틀려도 문 다시 열고 돌아가면 됨. → **빠르게, 소규모/개인이 결정.**
예: 대부분의 실험·기능 선택·UI·도구.

## 핵심 함정
조직이 커지면 *Type 2를 Type 1처럼* 무겁게 다뤄 느려진다. **되돌릴 수 있으면 빨리 결정하라.**

## 70% 규칙
90% 확신을 기다리지 마라. **70% 확신이면 결정하고 빠르게 수정**하는 게 낫다.

## Disagree & Commit
합의가 안 돼도, 결정 나면 다 같이 실행. 끝없는 논쟁보다 결정 + 실행 + 수정.
