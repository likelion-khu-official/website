# I (implementation) — 계획대로 만든다

계획을 코드로. 자율 구간이다(게이트 없음) — IQ 게이트까지 스스로 밀어붙인다. 매 스텝 사람에게 묻지 마라.

## 렌즈
- **계획을 따른다.** 구현 중 계획이 틀렸다고 느끼면 조용히 방향을 바꾸지 말고, 작은 조정은 하되 방향이 바뀌면 IQ 게이트에서 사유를 밝힌다(또는 큰 재설계면 RP로 되돌림).
- **읽기 쉽게.** 이름이 뜻을 말하게. 영리한 한 줄보다 지루한 세 줄(mindset). 다음 사람은 초보다.
- **기존 관례를 따른다.** 주변 코드 스타일·패턴에 맞춘다. 새 의존성·새 구조는 PM 게이트 → 함부로 추가 금지(막히면 ask-pm).
- **작게 커밋 가능한 단위로.** 각 변경이 하나의 Done에 대응하게 유지하면 Q·되돌림이 쉽다.
- **타입 명시, any 금지.** 타입 에러·버그는 자율로 고친다(팀의 학습).

## 경계
- Done 밖은 안 건드린다. "이왕 여는 김에" 리팩터 금지.
- 계약(`shared/`)을 바꿔야 하면 멈추고 확인 — 다른 팀 영향.

## 산출
동작하는 변경 + IQ 게이트에 낼 "만든 것 요약(어디에)". 곧바로 Q(셀프 QA)로.
