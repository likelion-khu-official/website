## [BE] — 피드(블로그) 백엔드: 글·댓글 API + 계약

콘텐츠 6기둥 중 **블로그는 글이 계속 쌓이는 유일한 동적 영역**이고, 그게 곧 "피드"예요(#59 기획, #58 설계 확정). 이번 미션은 그 데이터 레이어를 만드는 겁니다. 디자인이 화면을 그리고 프론트가 랜딩을 채우는 동안, 백엔드는 **계약을 먼저 열어** 병렬로 API를 세웁니다.

설계는 `#58`(확정사항)을 그대로 따르고, 배경은 `pm/docs/brief.md`를 참고하세요.

---

### Deliverable

- 글 API — 작성·목록·개별 조회, 상태 전이(draft → published → hidden)를 갖춘 저장소.
- 매직링크 토큰 — 로그인 없이 글을 쓰게 하는 1회용 토큰.
- 댓글 — 익명 반응 작성·조회 + 운영진 숨기기.
- 이미지 업로드 — Cloudflare R2 연동(인프라 #57 완료 후).
- `shared/types/feed.ts` — 프론트와 합의된 계약.

### Why it matters

> [!IMPORTANT]
> 이 프로젝트의 1순위 가치는 **참여 멤버의 실전 학습**이고, 피드는 "우리가 배우고 만든 이야기"를 밖으로 내보내 그 학습을 동아리 자산으로 바꾸는 장치예요. 지금 계약을 먼저 열어두면, 프론트가 나중에 블로그·피드 화면을 붙일 때 서로 기다릴 필요가 없습니다(핸드오프 릴레이가 아니라 병렬).

### Done

- [ ] 글을 **쓰고 → 검수해 공개하고 → 문제 시 숨기는** 흐름이 API로 동작한다 (공개 목록은 published만, 어드민은 전체)
- [ ] 매직링크 토큰이 1회용 + 24시간 만료로 동작한다
- [ ] 익명 댓글을 달고 볼 수 있고, 운영진이 숨길 수 있다
- [ ] `shared/types/feed.ts` 계약이 프론트와 합의돼 머지된다
- [ ] stage에서 **글 작성 → 발행 → 조회가 한 줄로** 실제로 된다

### Notes

- **R2(이미지)는 인프라 #57에 의존**해요 — 아직이면 이미지 업로드만 뒤로 미루고 글·상태·매직링크·댓글부터. R2 접근키는 찬욱님이 전달.
- 상태 전이 화이트리스트·`published_at` 보존 등 세부는 **#58 설계 확정사항** 그대로.
- **어드민 UI(검수 화면)는 이번 범위가 아니에요** — API까지. 운영진 인증은 모집 알림 쪽과 재사용할 수 있는지 보세요.
- 블로그와 피드는 같은 것으로 봅니다 — 화면도 하나로 통일할 예정이니, 계약이 프론트의 블로그 개별 글 화면에도 그대로 쓰이게 설계하세요.
