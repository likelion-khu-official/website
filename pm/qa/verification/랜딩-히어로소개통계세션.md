### LAND-001 — pass

`frontend/src/app/page.tsx:12-29`(`Home` 컴포넌트)가 랜딩의 전체 골격이다. `<main>` 안에 `Thumbnail → Introduce → IntroduceSession → Project → Members → Plan → Blog → Recruit` 순서로 컴포넌트가 나열돼 있고, 그 아래 `<Footer />`가 붙는다. 각 섹션의 `id` 속성으로 실제 렌더 순서를 다시 확인했다: `Thumbnail.tsx:4`가 `id="thumbnail"`(히어로), `Introduce.tsx:15`가 `id="introduce"`(소개+통계, 아래 참고), `IntroduceSession.tsx:61`이 `id="session"`(세션 소개), `Project.tsx:24`가 `id="project"`, `Members.tsx:126`이 `id="members"`, `Plan.tsx`(연간 활동계획)가 `id="plan"`, `Blog.tsx:50`이 `id="blog"`, `Recruit.tsx`가 `id="recruit"`. 이 순서가 GWT then에서 요구하는 "히어로→소개→통계→세션 소개→프로젝트→운영진→활동계획→블로그→모집→푸터"와 정확히 일치한다.

한 가지 짚을 점: "소개"와 "통계"는 원 시나리오 문구상 별도 섹션처럼 읽히지만, 실제 코드에서는 `Introduce.tsx` 하나의 컴포넌트(섹션) 안에 헤드라인(소개 카피, 12-77행)과 통계 카드(79-113행)가 함께 들어있다. DOM 순서상으로도 소개 카피가 먼저, 통계 숫자가 그 아래에 나오므로 then이 요구하는 "순서"는 깨지지 않는다 — 다만 별도의 `<section id="statistics">` 같은 건 없다는 점은 참고용으로 남긴다.

"서버 데이터 페치가 없어 로딩/에러 상태가 없다" 부분은 `grep -rn "fetch(\|axios\|useQuery\|async function\|await " frontend/src/components/sections/ frontend/src/app/page.tsx`로 확인했고, 랜딩을 구성하는 9개 섹션 컴포넌트와 `page.tsx` 어디에도 매치가 없었다. 즉 랜딩은 전부 하드코딩된 정적 데이터(예: `Blog.tsx:9-14`의 `DEFAULT_POSTS`, `Introduce.tsx:5-10`의 `stats` 배열)로 렌더되고, 서버에서 데이터를 받아오는 흐름 자체가 없으므로 로딩/에러 상태를 표시할 필요도, 코드도 없다.

### LAND-008 — pass

`frontend/src/app/layout.tsx:17-20`:

```ts
export const metadata: Metadata = {
  title: "멋쟁이사자처럼 at 경희대",
  description: "코딩으로 성장하는 경희대 개발 동아리",

```

title·description은 있지만 `openGraph` 필드는 이 객체에 없다 — 즉 루트에는 OG 태그가 없다. `robots.txt`·`sitemap.xml`(또는 Next.js의 `app/robots.ts`·`app/sitemap.ts`) 존재 여부는 `find frontend/src/app -iname "sitemap*" -o -iname "robots*"`와 `find frontend/public -iname "robots*" -o -iname "sitemap*"`로 확인했고 둘 다 결과 없음 — 루트 레벨에 robots.txt·sitemap이 없다.

"어드민 noindex 메타만 존재"는 `frontend/src/app/admin/layout.tsx:5`에서 확인: `robots: { index: false, follow: false }`. 즉 사이트 전체에서 검색엔진 색인 관련 메타는 어드민 레이아웃에만 있고, 공개 랜딩(루트)에는 없다. then의 세 조건(title/description 있음·OG 없음, robots.txt/sitemap 없음, 어드민 noindex만 존재) 모두 코드로 확인됐다.
