# 진행 로그 — #8 인프라 · stage/prod 추상화 (프론트가 붙는 서버 + 배포 뼈대)

> PM(김우진)이 **판단·결정한 것만** 시간순으로 추린다. GitHub 코멘트 복붙이 아니라, "왜 이렇게 됐나"의 정제된 흔적.
> 형식: `## YYYY-MM-DD` 아래 일어난 일 / 결정 / 왜.

## 2026-06-27 — 미션 설계 (stage의 주목적 재정의)

- **결정:** stage의 주목적 = "prod 리허설"이 아니라 **프론트가 개발 중 붙는 백엔드 서버**.
  → Done에 "프론트가 stage를 외부에서 호출(공개주소·CORS)"을 핵심으로 박음.
- **왜:** 프론트가 로컬 개발 중 mock 대신 실제 stage 백엔드에 연결해야 한 줄이 진짜로 물림.
  그게 안 되면 프론트가 막혀 — stage가 "항상 떠서 외부에서 닿는" 게 1순위.
- **결정:** 인프라가 stage/prod 복잡성을 **추상화**(개발팀은 "이렇게 올린다"만). + 장찬욱이 stage/prod
  개념을 모를 수 있어 미션에 *개념·왜*와 한국어 입문 자료를 담음 — 떠먹이지 않되 출발은 시킴.
- **결정:** 진짜 배포는 백엔드 의존이라, 이번엔 **더미 컨테이너로 틀**부터. 실앱은 백엔드 크면 합류.

## 2026-06-29 — CI/CD 독립 동작 확인 + 머지 정책 확정

- **확인:** dev(stage)·main(prod) CI/CD가 **독립적으로 정상 동작**. PR이면 테스트만, 머지면 실제 배포(백엔드 한정). 동작 정리는 PR #40. → stage가 "항상 떠서 외부에서 닿는다"는 미션 1순위가 실제로 성립.
- **결정(머지 정책):** **squash·rebase 금지, 일반 머지만 허용**(레포 설정). 왜 — dev→main을 squash 머지했더니 새 SHA로 히스토리가 갈려 공통 조상이 못 움직이고 머지가 막혔다. 오래 사는 브랜치는 공통 조상이 함께 전진해야 추적·롤백·다음 머지가 깔끔. 트레이드오프(feat 커밋이 다 남아 지저분) 받아들이고 *추적 가능*을 택함. → learnings 졸업.
- **결정(프론트 연결 = B안):** 프론트(Vercel)가 `/api/*`를 Next.js 서버사이드 rewrite로 백엔드에 프록시. 한 도메인(`likelion-khu.com`)만 노출, 백엔드 주소 은닉, CORS 불필요. 서브도메인 분리(A안) 대신. → 운영 사실은 `ops.md`, 패턴은 learnings.
- **상태:** 도메인 `likelion-khu.com` 확보(공용 명의). 스테이징은 `랜덤문자열.likelion-khu.com` 고정(깃 비노출). Vercel은 프론트 개인계정에 묶여 있어 *공용 이전 여부 미결*.
