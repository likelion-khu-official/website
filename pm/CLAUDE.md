# PM — 운영 하네스

> 김우진(PM·조직장)의 운영 베이스.

## 시작 전 읽기
1. 루트 `CLAUDE.md`
2. `pm/docs/`: `learnings.md` → `brief.md`

## PM 책임
- **결정** — 무엇을·왜. 방향·범위·우선순위.
- **운영** — 일정 조율, 막힌 것 해소, 스쿼드 통합. 진행은 GitHub 이슈(로드맵).
- **콘텐츠 수급** — 사진·소개글·자료를 운영진과 협의해 확보(전담 없음). 그림자 크리티컬 패스.
- **배운 것 관리** — *다음에도 쓸 통찰*이 생기면 `pm/docs/learnings.md`에. 결정·상태는 안 쌓음.
- **미션 박스 관리** — 미션 하나 = `pm/missions/<번호>-<슬러그>/` 상자(`proposal`→`log`→`result`). 제안·진행·결과의 정제된 맥락을 쌓는다(GitHub 복붙 아님). 운영은 mission 스킬.

## 무엇을 learnings에 남기나
- 남긴다: 다음에도 쓸 **통찰**(팀·제품·기술·과정에서 배운 것), 왜가 드러나게.
- 안 남긴다: 일회성 결정·기술 스택 선택·진행 상태·대화 과정. (진행은 GitHub)

## Git — 푸시 방식
- **김우진의 지시가 없는 한 PR을 올리지 않는다 — main에 직접 푸시한다.** PM 운영(하네스·문서·미션 박스)은 혼자 도는 작업이라 PR 게이트가 오히려 마찰. PR이 필요하면 김우진이 그때 명시한다.
- 단 **커밋·푸시 자체는 여전히 김우진의 지시가 있을 때만**(루트 협업 규약). "어떻게 올리나"가 직접 푸시일 뿐, "올릴까"는 지시 사항.
