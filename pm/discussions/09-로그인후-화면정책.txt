# 🔓 로그인 뒤 화면 — 멤버 · 관리자 · 최고관리자

> 🧭 **이 논의는 '수렴'하는 자리예요**
> 새로 벌리기보다, 정리해둔 이 안으로 함께 좁혀가는 자리예요. 의견 주기 전에 편하게 스스로 짚어봐 주세요:
> - 이건 **이 안에 대한 의견**일까요, 아니면 **새로 더하자는 제안**일까요?
> - (새 아이디어가 떠올랐다면 — 자연스러워요!) 그거 **지금 없으면 안 될까요, 나중에 더해도 될까요?**
> - 우리 규모(**부원 ~20명 · 글 월 2건**)에서 **정말 자주** 생기는 일일까요?
> - 기능으로 만들기 전에, **사람이 손으로** 해도 되지 않을까요?
> - 이 의견이 **지금 이 논의 주제 안**에 있을까요?

로그인하면 누구냐에 따라 화면이 갈려요. 일반 방문자는 이 화면들의 존재를 몰라요 (화면에 노출되지 않는, 즉 주소창 쳐서만 들어갈 수 있는 URL임)

## 못박고 가는 규칙
```
관리자    = 멤버 중 임명된 사람 (일반 멤버는 자동으로 관리자 아님)
           └ 현실의 동아리 '운영진' — 서비스 안에선 '관리자'로 불러요
최고관리자 = 딱 1명 (동아리 대표)
위임      = 위 → 아래로만 (최고관리자 → 관리자 / 관리자끼리 X)
```


## 🧑 멤버 화면 `/member` — 로그인한 부원 누구나
```
글쓰기
├─ 제목 · 본문 · 이미지 · 썸네일
├─ 임시저장 · 미리보기
└─ 제출 ─▶ 바로 게시 (검수 없음)
내 글 관리
├─ 수정
└─ 삭제
내 프로필 편집
├─ 사진 (넣으면 이모지 자리 대체)
└─ 입부계기
   ※ 이름·역할·기수·이모지는 관리자가 등록 → 본인은 못 고침
내 프로젝트
├─ 등록 (제목·소개·이미지 여러장+대표·참여멤버·기수·개발기간·스택·GitHub)
├─ 수정 (내가 참여한 것만)
└─ 삭제
```

## 🛡️ 관리자 화면 `/admin` — 임명된 관리자만
```
멤버 관리
├─ 등록 ─▶ 아이디=학번 · 초기비번=전화번호로 계정 발급
│          (이름·역할·기수 입력, 이모지는 자동 배정)
├─ 정보 수정 (옵셔널 — 이름·역할 정정, 부적절 시 사진·입부계기 손봄)
├─ 비밀번호 초기화 (분실 시)
└─ 오프보딩 ─▶ 소프트 딜리트 (로그인만 차단, 기록은 유지)
운영진 소개 관리
├─ 등록 · 수정 · 삭제
└─ 정렬 (회장 → 세션장 → 홍보부장 순서)
블로그 모니터링
├─ 글 숨김
└─ 댓글 숨김
프로젝트 모니터링
└─ 숨김
명단 열람
├─ 지원자 명단 (프론트가 JSON 파싱해서 표시)
└─ 모집알림 구독자
모집 관리
├─ 켜기 / 끄기 (지원폼 ↔ 모집알림 전환)
├─ 미리보기
└─ 구독자에게 안내 메일 발송 (모집 시작 시, no-reply@likelion-khu.com)
```

## 👑 최고관리자 화면 — 딱 1명
```
관리자 관리
├─ 임명 (멤버 → 관리자)
└─ 회수 (관리자 박탈 → 그 사람은 멤버가 됨)
최고관리자 이양 (승계)
└─ 다음 대표에게 넘김 → 넘기면 본인은 관리자로, 최고관리자는 늘 1명 유지
＋ 위 관리자 기능 전부
```
