## [BE] 운영진 소개 API — 운영진 카드 데이터 · 최고관리자 관리

랜딩의 **운영진 소개** 섹션에 뿌릴 데이터를 다루는 백엔드 API예요. 운영진 한 명을 **만들고, 고치고, 지우고, 목록으로 내려주는 것**까지요. 만들고·고치고·지우는 건 **최고관리자만** 할 수 있어요.

> [!IMPORTANT]
> **왜 필요한가.** 운영진 소개는 방문자가 "이 동아리를 실제로 누가 끌고 가나"를 보는 얼굴이에요. 지금 랜딩엔 이름도 직책도 플레이스홀더(`이름`, `Text Text…`)로 박혀 있어서, 운영진이 바뀔 때마다 개발자가 코드를 고쳐야 해요. 데이터로 빼두면 운영진이 직접 갱신하고, **매 기수 새 팀으로 얼굴을 바로 바꿀 수** 있어요.

### Data — 카드에 뭐가 들어가나

이 화면이 기준이에요. **멤버 API(#76)와는 다른 데이터**니까 헷갈리지 마세요(아래 Notes 참고).

**밖으로 보여지는 정보**

- **이름** — 실명이에요.
- **직책** — 회장·부회장·프론트 세션장·백엔드 세션장·기획/디자인 세션장·AI 세션장·홍보부장처럼요. **자유 텍스트**로 받아요(정해진 목록으로 가두지 않아요).
- **학과 / 학번** — 두 개 따로예요. 여기서 **학번은 입학 연도**를 말해요(예: `22`, `24` — "22학번·24학번"). 긴 학생번호가 아니에요.
- **사진** — 프로필 사진이에요. 이 카드의 비주얼 중심이라, 멤버 API랑 달리 **이모지 대체는 없어요.**
- **소개글** — 문단이에요(한 줄 아님). 회장·부회장 카드엔 소개 문단이 들어가고, 세션장·홍보부장 카드엔 없거나 짧아요. 그러니 **없어도 되게**요.
- **정렬값** — 화면에 뿌릴 순서예요. 프론트가 이 값 순서대로 카드를 놓고 계층(회장 → 세션장 → 홍보부장)을 표현해요.

**안에서만 쓰는 정보**

- **id**
- **created_by** — 누가 처음 등록했는지.
- **updated_by** — 누가 마지막으로 고쳤는지.
- **created_at / updated_at** — 만든·고친 시각이에요.

### API

- **운영진 등록** (최고관리자) — 위 정보를 받아 저장해요. 누가 만들었는지(`created_by`) 남겨요.
- **운영진 수정** (최고관리자) — 직책·사진·소개글·정렬값 등을 바꿔요. 누가 고쳤는지(`updated_by`) 남겨요.
- **운영진 삭제** (최고관리자) — 나간 운영진을 빼요.
- **운영진 목록** (아무나) — 운영진 전체를 **정렬값 순서로** 내려줘요.

### Definition of Done

- [ ] 최고관리자만 등록·수정·삭제를 할 수 있어요. 일반 운영진이나 로그인 안 한 사람은 막혀요.
- [ ] 운영진 목록을 아무나 조회할 수 있고, **정렬값 순서대로** 내려와요.
- [ ] 카드에 필요한 필드(이름·직책·학과·학번·사진·소개글·정렬값)가 응답에 다 담겨요.
- [ ] 사진을 올리고, 조회 응답에서 **사진 URL**로 받아와요.
- [ ] 등록엔 `created_by`, 수정엔 `updated_by`가 남아요.
- [ ] 응답 타입이 `shared/types`에 정의돼 있어요.

### Notes

> [!TIP]
> 관리 API의 '최고관리자' 접근은 **#74(어드민 인증)에서 오는 role에 기대요** — 멤버 API(#76)와 같은 방식이에요. #74가 선행이에요.

> [!IMPORTANT]
> **순서를 지켜주세요 — 계약(shared) 먼저.** `shared/types/`에 이 API의 요청·응답 타입부터 정의해서 **그것만 담은 작은 PR을 먼저** 올려주세요. 그 PR이 리뷰·머지되고 나서 나머지 구현(엔드포인트·저장 로직)을 이어가시면 돼요 — 계약이 먼저 확정돼야 프론트가 기다리지 않고 붙을 수 있어요.
> **이 PR들은 선우님 또는 시현님이 꼭 리뷰해주세요.** 백엔드 컨벤션(레이어 구조·에러 코드·응답 형태)은 두 분 머릿속에 있는 거라, 리뷰 없이 판단하지 말고 꼭 확인받고 머지해주세요. (지금 [#119 프로젝트 쇼케이스](https://github.com/likelion-khu-official/website/issues/119)도 같은 이유로 같이 진행 중이신 거 알아요 — 순서는 편하신 대로 정하세요.)

- **직책은 자유 텍스트, 정렬은 정렬값으로.** "회장 > 세션장 > 홍보부장" 같은 계층·시각 규칙은 프론트가 정렬값으로 표현해요. API는 **값과 순서만 정확히** 내려주면 돼요.
- **사진 저장은 새로 만들지 마세요** — 이미 있는 피드 이미지 업로드/스토리지(OCI) 인프라를 재사용하세요.
- **이건 멤버 API(#76)랑 별개예요.** 멤버 API는 전체 멤버 로스터(역할·기수·이모지), 이건 운영진 소개(직책·학과·학번·소개글)예요. 데이터를 합치거나 한쪽으로 대체하지 마세요.
- 어떻게 만들지(엔드포인트 경로·테이블·정렬 구현)는 찬욱님이 정하시면 되고, 애매한 부분은 선우님·시현님께 편하게 물어보세요.
