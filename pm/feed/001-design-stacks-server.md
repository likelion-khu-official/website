# 화면 설계가 나오고, 서버가 떴어요

이번엔 홈페이지를 만들 바탕을 깔았어요. 어떤 화면이 될지 전부 그려졌고, 그걸 무엇으로 만들지 정했고, 만든 걸 띄울 서버까지 준비됐어요. 아이디어만 있던 사이트가 이제 진짜로 만들기 시작할 준비를 끝냈어요.

## 어떤 홈페이지가 될지 그려졌어요

홈페이지의 모든 화면을 설계했어요. 소개·프로젝트·운영진·활동·블로그·모집까지 6개 페이지를 다 그렸어요. 메인 화면은 위에서 아래로 내려가며 동아리 세션(프론트엔드·백엔드·AI·디자인), 프로젝트, 운영진, 활동, 블로그, 지원하기가 순서대로 보이게 짰어요. 운영진 페이지는 카드를 나란히 놓는 안과 사진·글을 좌우로 번갈아 놓는 안 두 가지를 그려두고, 곧 하나로 정해요. 모집 알림 버튼은 평소엔 화면 구석에 작은 종으로 떠 있다가, 갖다 대면 '알림 받기'로 펼쳐지게 움직임까지 정했어요. 메인은 데스크탑과 모바일 둘 다 그렸고, 나머지 페이지의 모바일 화면과 색·글꼴이 다음 차례예요.

## 무엇으로 만들지 정했어요

이 화면들을 실제로 만들 도구를 정했어요. 화면 쪽은 Next.js로 — 검색에 잘 잡히고, 디자인 시안을 코드로 옮기기 좋고, 배포까지 매끄럽게 이어지는 조합이에요. 서버 쪽은 Java와 Spring으로 — 팀이 가장 빠르게 움직일 수 있는 선택이고, 무겁다는 걱정도 실제 서버 용량에 견줘 따져보고 문제없다고 확인했어요. 화면과 서버가 주고받을 데이터 형태도 미리 한곳에 맞춰뒀어요. 양쪽이 따로 만들다 모양이 어긋나 안 맞물리는 일을 처음부터 막아둔 거예요.

## 코드를 올리면 바로 인터넷에 떠요

그렇게 만든 걸 띄울 자리까지 미리 마련했어요. 24시간 도는 실제 서버를 올리고, 코드를 올리기만 하면 자동으로 그 서버에 새 버전이 배포되는 길을 깔았어요. 기능 하나를 완성하면 곧장 인터넷에 올라가요. 도커로 환경을 통일해서 누구 컴퓨터에서든, 또 서버에서든 똑같이 돌아가요. 이 서버는 월 0원이에요. 같은 사양을 다른 곳에서 빌리면 월 10만 원이 넘는 걸, 무료 한도 안에서 굴리도록 맞춰뒀어요.
