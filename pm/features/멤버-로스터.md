## 개요
- 멤버 탭(`/members`)의 멤버 카드 — 방문자 열람.
- 필드: 이름 · 역할(PM·FE·BE·디자인·인프라, 겸직 시 여러 개) · 기수(한번 정하면 불변) · 비주얼(이모지 또는 사진 딱 하나 — 처음엔 랜덤 이모지 자동, 본인이 사진 넣으면 대체) · 입부 계기 한 줄(없어도 됨).
- 뼈대(이름·역할·기수·이모지)는 관리자가 등록, 표현(사진·입부계기)은 본인이 채운다. 기수·이모지는 불변.
- 카드는 절대 비지 않는다(이모지 기본).

## 현재 진행 상태
백엔드 뼈대 API가 배포돼 동작한다 — `GET /api/members`(공개), `POST/PATCH /api/admin/members`(최고관리자, 랜덤 이모지 자동 배정·`created_by`/`updated_by` 기록, 기수·이모지 불변)와 `shared/types/member.ts` 계약이 PR #98로 머지됐고, PR #101에서 `created_by` 해시값·타임스탬프 저장 버그가 잡혔다. 다만 스펙의 "본인이 사진·입부계기를 편집"하는 멤버 셀프 편집 경로는 아직 없다(현재 편집은 최고관리자 전용 PATCH뿐이며, 멤버 로그인 구조는 PR #97에서 미확정으로 남음). 프론트·디자인은 미완 — 디자인 이슈 #77이 아직 OPEN이다.
