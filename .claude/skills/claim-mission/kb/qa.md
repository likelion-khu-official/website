# Q (qa) — Done을 하나씩 증명한다

"빌드 됨/테스트 통과"는 완료가 아니다. **미션의 `Done` 각 항목이 참임을 근거로 보이는 것**이 완료다.

## 렌즈
- **Done 체크박스별로 증거.** 항목마다: 무엇으로 확인했나(테스트 실행 결과·앱 실제 구동·화면·API 응답). "됐을 것"이 아니라 "이렇게 확인했다".
- **실제로 굴려본다.** 코드를 읽는 것과 도는 것은 다르다. 바꾼 흐름을 실제로 실행/렌더해 관찰. (테스트·문서만 바꾼 게 아니면 반드시.)
- **Notes 경계 재확인.** 하지 말라던 것 안 했나, 다른 팀 접점 안 깼나.
- **스코프 크리프 감사.** Done에 없는 변경이 diff에 있나? 있으면 덜어내거나(원복) IQ 게이트에서 사유를 밝힌다.
- **초보 관점 한 번.** 다음 사람이 이 변경을 5분에 이해하나(mindset).

## IQ 게이트에 낼 것
Done 항목별 참/근거 표 + Notes 경계 확인 + 스코프 크리프 없음. 사람이 승인하면 **채간 사람이 Step 4에서 마감**한다 — `close-mission.sh`로 보드 Done + 이슈 close. (미션은 연 사람이 닫는다 — `contract.md`.)

## 함정
- 자기 코드에 관대하기 → Done을 반증하려 해봐라(안 되는 케이스 먼저 쳐봐).
- Done을 못 채웠는데 "대충 됨"으로 넘기기 → 되돌림(I로)이 정답. Done이 진실.
